## Описание проекта

![Главная страница](Главная.png)

Данный проект представляет собой веб-приложение для управления персоналом, предназначенное для упрощения работы с данными сотрудников. Система позволяет:
- Просматривать, добавлять, редактировать и удалять записи о сотрудниках.
- Управлять ролями пользователей (Admin, Editor, User) с разграничением прав доступа.
- Анализировать статистику по сотрудникам (возраст, пол, образование, должность и др.).
- Работать с данными через удобный веб-интерфейс или API.

Приложение поддерживает аутентификацию и авторизацию на основе JWT, а также интеграцию с базой данных MySQL для хранения информации.

## Установка и запуск

### Запуск через Docker

#### Предварительные требования
- Установленные Docker и Docker Compose.

#### Инструкции по запуску
1. Перейдите в директорию проекта:
   ```bash
   cd d:\projects\HR
   ```

2. Соберите и запустите контейнеры:
   ```bash
   docker-compose up --build
   ```
   Для запуска в фоновом режиме используйте флаг `-d`:
   ```bash
   docker-compose up --build -d
   ```

3. После успешного запуска:
   - Приложение будет доступно по адресу: [http://localhost:3000](http://localhost:3000)
   - База данных MySQL будет доступна на порту `3307`.

4. Для остановки контейнеров выполните:
   ```bash
   docker-compose down
   ```

### Запуск вручную

#### Предварительные требования
- Установленные Node.js и MySQL.

#### Инструкции по запуску
1. Установите зависимости:
   ```bash
   npm install
   ```

2. Убедитесь, что MySQL запущен и доступен. При необходимости настройте подключение в `src/config/db.js`.

3. Запустите сервер:
   ```bash
   node server.js
   ```

4. После успешного запуска:
   - Приложение будет доступно по адресу: [http://localhost:3000](http://localhost:3000)
   - База данных MySQL будет доступна на порту `3306` (или другом, если настроено иначе).

## Система входа и регистрации

### Роли
В приложении реализована система ролей с тремя уровнями доступа:

- **User** — только просмотр данных.
- **Editor** — редактирование существующих данных.
- **Admin** — полные права на добавление, удаление и изменение данных.

### Демонстрационные аккаунты
В базе данных по умолчанию доступны три аккаунта:

| Логин  | Роль    | Пароль     |
|--------|---------|------------|
| admin  | admin   | admin123   |
| user   | user    | user123    |
| editor | editor  | editor123  |

### Регистрация
Пользователи могут создать свой аккаунт, указав логин, пароль и желаемую роль. Для ролей **Editor** и **Admin** требуется служебный пароль:

- Служебный пароль админа: `adminsecret`
- Служебный пароль редактора: `editorsecret`
